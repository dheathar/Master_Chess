What Is Built & Working

✓ BKT Skill Tracking
Bayesian Knowledge Tracing per concept tag (bkt.ts). SM-2-lite spaced repetition for review scheduling (reviewSchedule.ts).
✓ Socratic AI Coach
Turn-taking coach with Socratic questioning, dialogue memory, and misconception recovery. Graded outcomes update the learner model.
✓ Multiple Coach Archetypes
Five archetypes (Socratic, Warm Mentor, Drill Sergeant, Playful, Analytical) with tunable verbosity, tone, and jargon sliders via Coach Sculptor.
✓ Teacher Classroom Management
Teacher Dashboard manages student roster, assigns coaches and lessons, tracks per-student progress (games, puzzles, lessons, streak, ELO).
✓ Stockfish Engine Integration
Cloud Stockfish oracle (agent/oracle/stockfish.ts) grounds all analysis. Coaching narratives cannot contradict engine verdicts.
✓ Game Analysis with Commentary
Analyze mode with engine-grounded step-by-step commentary (analysisCommentary.ts). Move-level insights delivered via coaching voice.
✓ Six Game Modes
Play vs AI · Watch (Atlas vs Counterplay) · Analyze · Lesson · Diagnostic · Agent Self-Play — all accessible from the student dashboard.
✓ Curriculum Planner
Curriculum graph (curriculumGraph.ts) and planner (curriculumPlanner.ts) — concept dependency tree with BKT-aware sequencing.
✓ Semantic Dialogue Memory
Per-session and cross-session dialogue memory enables the coach to refer back to prior mistakes and correct recurring misconceptions.
✓ Position Substrate System
Richly annotated positions with concept tags, hint ladders, candidate move judgments, and level-appropriate explanations (positions.ts).
✓ Historic Games Reference
Curated master game corpus (historicGames.ts) used for coaching illustration. Misconceptions library maps common errors to instructional replies.
✓ Teacher Knowledge Studio
Chat-based coach for teachers to explore chess concepts and encode knowledge into the platform (teacherKnowledge.ts, teacherKnowledgeCoach.ts).

✕
PGN upload / import from Chess.com or Lichess
This is the pitch's primary entry point — "Upload any PGN." Without it the self-service product loop cannot begin. The current system works on curated positions, not player-submitted games.
Critical
✕
Per-move engine evaluation for full game (blunder / mistake / inaccuracy)
SRS §3.4 requires Stockfish eval at every move. The roadmap item "Move-by-Move Analysis (€20k, 4 weeks)" directly targets this. Currently Stockfish is called for curated positions only.
Critical
✕
Separate White / Black analysis per game
Investor pitch slide 3 promises "Separate analysis for White and Black, rated high/medium/low." Not yet implemented — all current analysis is position-level, not player-color-level.
Critical
Player Model — Skill Taxonomy & Level Framework §3.1
✕
23-skill taxonomy from PlayerFramework (currently 9 concept tags)
The PlayerFramework.xlsx defines 23 skills across 4 categories (Opening, Middlegame, Endgame, Psychology) with sub-categories. The current model has 9 tactical concept tags (fork, pin, back-rank, etc.) — covering only a narrow slice of the full framework.
Critical
✕
7-level player classification (L1 <800 through L7 2300+)
The framework maps every skill requirement to a rating level. The current model uses beginner / intermediate / advanced — too coarse to drive the level-appropriate prescriptions the SRS requires.
High
✕
Plateau diagnosis (5 named plateaus from PlayerFramework)
Framework defines five diagnosable plateaus: Blunder Wall (<1200), Strategy Desert (1400–1600), Conversion Ceiling (1500–2000), Prophylaxis Gap (~1800), Precision Boundary (2000–2200). None are implemented. The SRS acceptance criterion explicitly requires the dashboard to "name the current plateau."
High

BKT currently updates from drill answers. The SRS requires it to update from every imported game's engine-grounded evidence. This is blocked by the PGN pipeline gap above.

♛ Strategic Note — What the Codebase Already Is

The existing system is architecturally coherent and technically sound — it is a teacher-managed classroom coaching platform with sophisticated pedagogical machinery (BKT, SM-2-lite, Socratic agent loop, engine grounding, six play modes). That is genuinely valuable and directly maps to the Coach/Academy tier (€29–49/mo) of the business model.

The gap between the codebase and the SRS vision is not architectural disorder — it is a persona gap. The SRS describes a self-service individual player who arrives with their own Chess.com PGN file and receives an autonomous diagnostic. The codebase assumes a teacher who sets up students and assigns them work. Bridging this gap means adding the PGN ingestion pipeline (§3.4) and wiring it to the full 23-skill Player Model (§3.1) — the rest of the features follow from those two anchors.

The Beta milestone (€20k, 4 weeks) maps precisely to this: per-move engine evaluation on uploaded PGNs. That single capability, combined with a progress dashboard showing skill trajectory and plateau diagnosis, would take the product from ~35% to ~65% of the SRS vision and make the Beta investor demo compelling.
